Question: Outline the steps you took to get to your final CNN architecture and your reasoning at each step. Describe why you think the architecture is suitable for the current problem.

Answer:

I used the ResNet model, specifically ResNet-18, which is pretrained. Our dataset is about landmark classification, and I believe this architecture is suitable for transfer learning in our case because the ResNet model was initially trained on ImageNet, a dataset similar in nature to our own dataset (Landmark classification).

About ImageNet:

  • The ImageNet dataset is used to train the ResNet model.
  • ImageNet is a large, well-annotated collection of millions of labeled images across thousands of categories.

To adapt ResNet-18 for our specific problem, I replaced only the head (the fully connected layer) of the model. I then trained this part of the model exclusively. Given that the pretrained model has been trained on data similar to ours, it was sufficient to train only the classifier without fine-tuning the backbone. This approach allowed us to achieve a 67% accuracy on the test set with just 20 epochs.

from src.predictor import Predictor
from src.helpers import compute_mean_and_std

# First let's get the class names from our data loaders
class_names = data_loaders["train"].dataset.classes

# Then let's move the model_transfer to the CPU
# (we don't need GPU for inference)
model_transfer = model_transfer.cpu()
# Let's make sure we use the right weights by loading the
# best weights we have found during training
# NOTE: remember to use map_location='cpu' so the weights
# are loaded on the CPU (and not the GPU)
model_transfer.load_state_dict(
    torch.load("checkpoints/model_transfer.pt", map_location="cpu")['model_state_dict']
)

# Let's wrap our model using the predictor class
mean, std = compute_mean_and_std()
predictor = Predictor(model_transfer, class_names, mean, std).cpu()

# Export using torch.jit.script
scripted_predictor = torch.jit.script(predictor)
scripted_predictor.save("checkpoints/transfer_exported.pt")
